1. What are the different types of HTTP requests

    HyperText Transfer Protocol(HTTP) specifies a collection of request method which speicifies what action is to be performed on a particular resource. The most commonly used http request methods are GET, POST, PUT and DELETE.

    • GET : this request is used to read data from a web server. The GET request returns an HTTP Status code of 200(request is successful) if the data is successfully retrieved from the server.
    • Post : this request is used to send the data to the server. It can only be retrieved in the body. It is used when we need confidential data as it is only retrievable at the backend and not visible on the webpage. On successful creation, it returns an HTTP statu code 201
    • PUT : this request is used to modify the data on the server. it replaces the entire content at a particular location with the data that is passed in the body payload. If there are no resources that match the request, it will generate one.
    • DELETE : this request is used to delete the data on the server at a specified location.

  2. Explain the concept of middleware in Nodejs

    Express.js is a routing framewaork used for managing different routing of the webpage and it works between the request and response cycle. A middleware comes in between a request and a response. Middleware gets executed after the server recieves the request and before the controller actions send the response. The Middleware has the access to the request object, response object and the next() function, it can process the request before the server and send a response. Middlewares were introduced to avoid DDOs attack. DDOs stands for Direct Denial of Service, is an attack which occurs when a server recieves thousands or millions of requests in a short period od time, and the serves does not have sufficient connections(ports) to fulfill all the requests and the server may crash. Thus, the middleware was introduced as if the server is full, the middleware can decline your request unless there is a connection available in the server.

  3. Explain CORS

    CORS : Cross Origin Resource Sharing is a mechanism by which the data or any resource of a site can be shared intentionally to a third party website when there is need. Every server runs on different platforms, and any server cannot connect with another server just like that, there has to be medium through which they can connect with one another. CORS also acts as a middleware and can also help us redirect users to a specific URL for eg. if I want to register to Linkedin through my gmail account, CORS can redirect me to google url to help me log into my google account and create my Linkein account. Generally, access to rescources that are residing in a third party site is restricted by the browser clients fot security pruporses.

  5. Explain 5 status codes that are used in Backend development

    A status code is a message the website servers sends to the browser to indicate whether or not that request can be fulfilled. Status codes specs are set by the W3C. Status Codes are embedded in the HTTP header page to tell the browser the result of its request

    • *102 - Processing
    • *200 - OK
    • *400 - Bad request
    • *402 - Payment Required
    • *403 - Forbiden
    • *404 - Not Founf/Error
    • *500 - Internal server error
    • *503 - Service unavailable

  6. Difference between HTTP and HTTPS

    The only difference between these 2 protocols is that HTTPS used TLS(SSL) to encrypt normal HTTP requests and responses. As a result, HTTPS is far more secure thatn HTTP. The website that uses HTTP has http:// in its URL and a website with HTTPS has https:// in its website.
